# 🎯 My Learning Methodology

To transition from a developer to an elite problem solver, I follow a strict **"Build, Break, Optimize"** feedback loop:

## 1️⃣ Active Code-Along & Commentary
Every line of code is written concurrently with the lessons, heavily documented with custom comments explaining **why** a particular approach was taken.

## 2️⃣ First-Principles & Manual Dry-Running
Prioritizing tracing code on paper. I sketch pointer movements, recursive call trees, and variable tables before committing solutions to code.

## 3️⃣ Complexity-First Architecture
Analyzing and documenting the strict **Time Complexity (`O`)** and **Space Complexity** for every implementation to ensure optimal execution performance.

## 4️⃣ Targeted Interview Application
Consistently solving assignment tasks on:
- LeetCode
- GeeksforGeeks
- Codeforces

This helps reinforce conceptual patterns and interview-level thinking.
